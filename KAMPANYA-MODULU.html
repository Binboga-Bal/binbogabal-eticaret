Kampanya Motorunun Katmanları

  1. Aktif kampanyaları yükle (5 dk cache)
  2. Kupon kodu varsa → kupon kampanyasını bul
  3. Koşulları değerlendir (AND/OR grupları)
  4. Segment eşleştir (müşteri profili)
  5. Çakışma çöz (stackable, öncelik)
  6. Aksiyonları çalıştır (indirim hesapla)
  7. Sonucu döndür (AppliedCampaign[])

Güvenlik Katmanları

  • Tüm kampanya hesaplamaları sunucu tarafında yapılır
  • Sipariş oluştururken kampanya yeniden doğrulanır
  • Kupon kodu: 10 deneme / 15 dk rate limit
  • Tüm /api/admin/* route'ları JWT ile korunur
  • Cron job'lar CRON_SECRET Bearer token ile korunur
  • requiresApproval: true kampanyalar ACTIVE olamaz
  • Her değişiklik CampaignAuditLog'a kaydedilir

Model Açıklamaları

Campaign.status akış şeması

DRAFT
→
PENDING_APPROVAL
→
APPROVED
→
ACTIVE
→
PAUSED / ENDED / CANCELLED

Cron campaign-activate dakikada bir çalışır: startsAt ≤ now olan APPROVED kampanyaları → ACTIVE.
Cron campaign-expire: endsAt < now olan ACTIVE kampanyaları → ENDED.

Campaign.stackable Mantığı

  • stackable: false — bu kampanya uygulandıktan sonra diğer non-stackable'lar durdurulur
  • stackable: true — diğer kampanyalarla birikimli çalışır
  • Kupon + otomatik indirim: her ikisi de stackable ise birlikte uygulanır

Campaign.budgetLimit Mantığı

  • Her kullanımda budgetUsed artar
  • Cron campaign-budget-check saatlik çalışır
  • budgetUsed ≥ budgetLimit olunca → PAUSED
  • Sipariş oluşturulunca CampaignUsage kaydedilir

// Çağrı noktası: POST /api/campaigns/evaluate
async function evaluateCampaigns(ctx: EvaluationContext): Promise<CampaignResult>

interface EvaluationContext {
  cart: CartItem[]          // sepet öğeleri (categoryIds sunucuda zenginleştirilir)
  customer: Customer | null // oturum açık kullanıcı profili
  couponCode?: string       // girilen kupon kodu
  device?: "mobile" | "desktop"

interface CampaignResult {
  appliedCampaigns: AppliedCampaign[]
  totalDiscount: number
  freeShipping: boolean
  giftProducts: GiftProduct[]
  cashbackPoints: number
  messages: string[]        // kullanıcıya gösterilecek mesajlar
}

Cache Stratejisi

  • Aktif kampanyalar in-memory cache'de tutulur (5 dakika TTL)
  • Kampanya oluşturulunca / güncellenince / durumu değişince invalidateCampaignCache() çağrılır
  • Redis varsa ileride cache layer değiştirilebilir — interface aynı kalır

Koşullar (IF) — condition-evaluator.ts

AND / OR Logic Group Mantığı

Her koşulun logicGroup (0, 1, 2…) alanı vardır:

  • Aynı grup içindeki koşullar AND ile bağlıdır
  • Farklı gruplar OR ile bağlıdır
  • Herhangi bir grubun tüm koşulları sağlanırsa kampanya uygulanır
// Örnek: "500 TL üzeri VEYA VIP müşteri ise indirim uygula"
conditions = [
  { type: "MIN_CART_AMOUNT", value: { amount: 500 }, logicGroup: 0 },
  { type: "CUSTOMER_SEGMENT", value: { segment: "VIP" }, logicGroup: 1 }
]

Aksiyonlar (THEN) — action-executor.ts

BUY_X_PAY_Y Hesaplaması

Sepetteki toplam ürün sayısına göre en ucuz ürünler ücretsiz sayılır:

const freeCount = Math.floor(totalQty / buyX) * (buyX - payY)
// Örnek: 6 ürün, 3 al 2 öde → 2 ürün bedava
// En ucuz 2 ürün fiyatı totalDiscount'a eklenir

maxDiscountAmount Tavanı

Her kampanyanın maxDiscountAmount alanı hesaplanan indirimi tavanda keser:

discount = Math.min(calculatedDiscount, campaign.maxDiscountAmount)

Çakışma & Öncelik — conflict-resolver.ts

Öncelik Kuralları

  1. priority değeri büyükten küçüğe göre sıralanır
  2. stackable: false bir kampanya uygulandıktan sonra diğer non-stackable kampanyalar durdurulur
  3. stackable: true kampanyalar birikimli uygulanmaya devam eder
  4. maxDiscountAmount her kampanya için bağımsız tavandır
  5. budgetLimit dolunca kampanya otomatik PAUSED'a alınır

Örnek: Yaz İndirimi (priority=10, stackable=false) + Ücretsiz Kargo (priority=5, stackable=true) → Her ikisi uygulanır, Yaz İndirimi sonrası başka non-stackable kampanya çalışmaz.

A/B Test Motoru — ab-router.ts

Deterministik Yönlendirme

Müşteri ID veya session bazında her zaman aynı varyanta yönlendirilir:

// seed = müşteri ID (giriş yapmışsa) veya rastgele string
let hash = 0;
for (let i = 0; i < seed.length; i++) {
  hash = (hash * 31 + seed.charCodeAt(i)) % 100;
}
// trafficSplit'ler kümülatif toplanarak varyant seçilir

Varyant Yönetimi

  • Her varyant: variantName, trafficSplit (%), discountValue (%)
  • Tüm trafficSplit değerleri toplamı = 100 olmalı
  • isWinner: true olan varyant kazanan olarak işaretlenir
  • Metrikler: impressions, conversions, revenue (manuel güncelleme veya future analytics)

Rate Limiter — rate-limiter.ts

Kupon Kodu Brute-Force Koruması

  • IP başına 10 deneme / 15 dakika penceresi
  • In-memory Map kullanır (sunucu restart'ta sıfırlanır)
  • Başarılı doğrulamada IP sayacı sıfırlanır
  • 429 yanıtında X-RateLimit-Remaining: 0 header'ı döner

CartStore Yeni State

campaignResult: CampaignResult | null
setCampaignResult: (r) => void

// total() artık kampanya indirimini bilir:
total: () => {
  const discount = campaignResult
    ? campaignResult.totalDiscount
    : couponDiscount; // geri dönüş
  return Math.max(0, subtotal - discount);
}

Kupon → Kampanya Motoru Köprüsü

  • Kullanıcı kupon kodu girer
  • /api/campaigns/coupon/validate ile hızlı doğrulama
  • Geçerliyse applyCoupon() → couponCode store'a yazılır
  • useCampaignEvaluator yeniden tetiklenir (couponCode dependency)
  • Evaluate endpoint kupon kampanyasını da bulur ve uygular

Sipariş Oluşturulurken

  • CheckoutForm → appliedCampaignIds[] ve effectiveDiscount ile sipariş gönderir
  • POST /api/orders → her kampanya için CampaignUsage kaydı oluşturur
  • Her kampanyanın budgetUsed güncellenir
  • Kupon usedCount artırılır (eski mekanizma korundu)

Güvenlik Notları

  • Client manipülasyonuna karşı: Sepet indirimleri her zaman sunucu tarafında yeniden hesaplanır. Frontend yalnızca gösterim için sonucu kullanır.
  • Admin koruması: Tüm /api/admin/* route'ları session role kontrolü yapar (ADMIN/SUPERADMIN/EDITOR).
  • Kupon brute-force: /api/campaigns/coupon/validate → 10 deneme / 15 dk rate limit (IP bazlı).
  • Cron koruması: Authorization: Bearer CRON_SECRET header zorunlu.
  • Onay akışı: requiresApproval: true kampanyalar ACTIVE olamaz, önce SUPERADMIN onayı gerekir.
  • Audit log: Admin tarafından yapılan her değişiklik (oluşturma, güncelleme, onaylama, duraklatma) CampaignAuditLog'a IP ile birlikte kaydedilir.
