Genel Bakış

Proje Hakkında

Binboğa Bal, 1973 yılında Adana'nın Kozan ilçesinde kurulan kooperatifin modern e-ticaret platformudur. Platform; ürün yönetimi, sipariş takibi, müşteri hesapları, blog/içerik yönetimi ve admin panelini kapsar.

Temel Hedefler

  • Marka kimliğini yansıtan özgün bir tasarım (bal/petek motifli)
  • Tüm renk ve boyut değerlerini tek dosyadan yönetmek (lib/theme.ts)
  • Performanslı ve SEO dostu Next.js App Router mimarisi
  • Kooperatif anlatısını ön plana çıkaran içerik yapısı
  • Mobil öncelikli responsive tasarım

Merkezi Tema Sistemi — lib/theme.ts

Problem

Çözüm: Tek Kaynak (Single Source of Truth)

  • §3 sliderTheme — Slider içerikleri, hız, yükseklik
  • §4 buttonTheme — Buton renk seti
  • §5 adminTheme — Admin panel renkleri
  • §6 tailwindColors — Tailwind'e aktarılan renk nesnesi
  • §7 — CSS değişkenleriyle senkron notu
  • §8 hexToRgba() — Yardımcı fonksiyon

    }
  }

Header Tasarımı — SVG Arch

Tasarım Kararı

Standart dikdörtgen header yerine, ortada aşağı doğru kavisli (arch/wave) bir header tasarlandı. Logo, bu kavisin tam ortasına yerleştirildi. Tasarım SVG ile çizildi — hiçbir CSS gradient veya clip-path kullanılmadı. 

/*
  Header şekli (temsili):
  ┌──────────── TAM GENİŞLİK ────────────┐  ← navHeight (85px)
  │  [ÜRÜNLERİMİZ]  [LOGO]  [HAKKIMIZDA] │
  └──────────╮               ╭───────────┘
              ╲      🍯      ╱
               ╰─── arch ───╯  ← waveDepth (90px) kadar iner
*/

// SVG path hesaplaması
const VB_W = 1440;
const VB_H = headerTheme.navHeight + headerTheme.waveDepth; // 85+90=175

// Beyaz dolgu path'i
const headerPath = [
  `M 0 0`,
  `L 1440 0`,
  `L 1440 85`,
  `L 900 85`,
  `C 820 85, 780 175, 720 175`,  // sağdan merkeze bezier
  `C 660 175, 620 85, 540 85`,   // merkezden sola bezier
  `L 0 85`,
  `Z`,
].join(" ");

Navigasyon Yapısı

  • Sol nav: yalnızca ilk link (ÜRÜNLERİMİZ)
  • Orta: logo için boşluk (logoWidth + 48px)
  • Sağ nav: kalan linkler + User / Search / Cart ikonları
  • Mobil: hamburger menü, arama ve sepet ikonları

Wave / Arch Görsel Düzeltmeleri

Problem: Arch Kesik Görünüyordu

Arch'ın en alt noktasındaki çizgi (waveStroke), SVG viewBox sınırında yarıya kesiliyor; arch "tıraşlanmış" görünüyordu.

✓ ÇÖZÜM: SVG elementine overflow="visible" eklendi. Böylece stroke viewBox sınırında kırpılmıyor. 
<svg
  viewBox="0 0 1440 175"
  overflow="visible"          ← bu satır eklendi
  preserveAspectRatio="none"
  className="absolute inset-0 w-full h-full"
>
  <path d={headerPath} fill="white" />
  <path
    d={archBorderPath}
    fill="none"
    stroke="#C57930"
    strokeWidth="6"
  />
</svg>

Wave Derinliğini Artırma

Kullanıcı isteğiyle wave daha derine indirildi, slider/sayfa görselleri üzerine binmesi sağlandı. waveDepth değeri 60 → 90px'e artırıldı.

Shop Layout & Header–İçerik Örtüşmesi

Hedef

Header'ın arch/wave kısmı, hero slider ve bölüm görsellerinin üstüne binmeli. Ama aynı zamanda, sayfa içeriği headerın arkasına gizlenmemeli.

Çözüm: fixed + paddingTop Stratejisi

  • Header: position: fixed → akıştan çıkar, her zaman üstte kalır
  • <main>: paddingTop = solidHeight (125px) → içerik "solid" bölümün altından başlar
  • Wave (90px): main'in padding'inin üzerinden içeriğe biner — efekt sağlanır
// app/(shop)/layout.tsx
import { headerTheme } from "@/lib/theme";
const MAIN_PT = headerTheme.solidHeight; // 125 px

export default function ShopLayout({ children }) {
  return (
    <>
      <Header />
      <main style={{ paddingTop: MAIN_PT }}>{children}</main>
      <Footer />
    </>
  );
}
⚠️ BAKIM NOTU: solidHeight = announcementHeight + navHeight. Bu değerlerden biri değiştiğinde solidHeight de manuel güncellenmeli!

Hero Slider — Swiper.js ile İmaj & Gradient Modu

Problem: Geçişler Kötü Görünüyordu

Sadece görsel tanımlandığında geçişlerde beyaz flaş oluşuyor, üzerinde istenmeyen overlay/fade görünüyordu.

Çözüm: EffectFade + crossFade + Dual Branch

  • effect="fade" + fadeEffect={{ crossFade: true }} → geçişlerde arka planın görünmesini önler
  • Image branch: slide.image varsa → sadece tam ekran görsel, hiç overlay yok
  • Gradient branch: slide.image yoksa → gradient arka plan + metin + istatistikler
function SlideContent({ slide }) {

  // ── IMAGE ONLY BRANCH ──────────────────────────────────
  if (slide.image) {
    return (
      <section className="hero-slide-section relative overflow-hidden bg-black">
        <img
          src={slide.image}
          className="absolute inset-0 w-full h-full object-cover"
        />
        </* Alt dekoratif dalga SVG */>
      </section>
    );
  }

  // ── GRADIENT + TEXT BRANCH ─────────────────────────────
  return <section style={{ background: `linear-gradient(...)` }}>...</section>;
}

Yükseklik Sorunu: Swiper Fade'de Container Collapse

Fade modunda slide'lar position: absolute olduğu için Swiper container'ı JS çalışana kadar height: 0'a düşüyor; beyaz alan görünüyordu.

✓ ÇÖZÜM: Hem wrapper div'e hem de <Swiper>'a style={{ minHeight: sliderTheme.height }} eklendi. Responsive yükseklik için CSS @media kuralları da eklendi. 
// Responsive yükseklik — theme değerlerinden
<style>{`
  .hero-slide-section { min-height: ${sliderTheme.heightMobile}px; }
  @media (min-width: 768px) {
    .hero-slide-section { min-height: ${sliderTheme.height}px; }
  }
`}</style>

// sliderTheme.slides içeriği theme.ts'ten gelir
const slides = sliderTheme.slides as unknown as Slide[];

TypeScript: as const Union Sorunu

as const ile tanımlanan slides array'indeki literal union tipler gradientFrom? gibi optional alanları bulamıyordu. 

// ❌ Çalışmayan yaklaşım
type Slide = (typeof sliderTheme.slides)[number];

// ✅ Çalışan çözüm
interface Slide {
  primaryBtn:   { label: string; href: string };
  secondaryBtn?: { label: string; href: string };
  image?:       string;
  badge?:       string;
  title?:       string;
  // ... tüm alanlar optional
}
const slides: Slide[] = sliderTheme.slides as unknown as Slide[];

Wave SVG — Baseline Gap Düzeltmesi

Sorun

Slider alt kısmındaki dalga SVG elementi display: inline (tarayıcı varsayılanı) olarak render ediliyordu. Inline elementlerde tarayıcı, metnin alt çizgisine (baseline) hizalama için SVG'nin altına 2–4 px boşluk ekler — bu boşluk ince yatay çizgi olarak görünüyordu.

Wave-Image — Subpixel Gap Düzeltmesi

Sorun

Wave SVG konteyneri absolute bottom-0 ile section'ın tam dibine konumlandırılmıştı. Farklı viewport genişliklerinde SVG yüksekliği kesirli piksel hesaplamalar üretebilir (örn. 56.916…px → 57px). Bu yuvarlama farkı, wave ile image/section tabanı arasında 1–2 px'lik görünür boşluk bırakıyordu.

Çözüm

Wave konteyneri bottom: -2px ile section sınırının 2 px altına taşındı. Section'ın overflow-hidden'ı bu taşmayı keser; görünür alt kenar her zaman section tabanıyla örtüşür, boşluk kalmaz. 

/* Önceki */
<div className="absolute bottom-0 left-0 right-0 pointer-events-none">

/* Sonraki — 2 px taşma, overflow-hidden tarafından kesilir */
<div
  className="absolute left-0 right-0 pointer-events-none"
  style={{ bottom: '-2px' }}
>
⚠️ Ön Koşul: Section elementinde overflow-hidden olması zorunludur; aksi hâlde wave 2 px aşağı sarkar ve alttaki bölümle örtüşür.
✅ Sonuç: Wave ile image arasındaki ince çizgi tüm viewport genişliklerinde giderildi.

Çözülen Hatalar

1 — SVG Stroke Kırpılması

Belirti: Arch alt kenar çizgisi yarıya kesik görünüyordu.

Neden: SVG viewBox sınırında stroke varsayılan olarak kırpılır.

Çözüm: <svg overflow="visible">

2 — Swiper Fade'de Container Yüksekliği Çöküşü

Belirti: Sayfa yüklenirken beyaz boşluk görünüyordu.

Neden: Fade modunda slide'lar position: absolute → container 0px yüksekliğe düşüyor.

Çözüm: Hem wrapper'a hem <Swiper>'a style={{ minHeight: sliderTheme.height }}

3 — TypeScript as const Union Type Hatası

Belirti: Property 'gradientFrom' does not exist on type

Neden: as const her slide'ı ayrı literal union olarak tanımlıyor; ortak alanlar bulunamıyor.

Çözüm: Explicit interface Slide + as unknown as Slide[] cast

4 — solidHeight Yanlış Hesabı

Belirti: İçerik header'ın arkasına giriyordu.

Neden: announcementHeight ve navHeight güncellendi ama solidHeight eski değerde kaldı.

Çözüm: solidHeight güncellendi (108 → 125). Uyarı yorumu eklendi.

5 — JSX Yorum Parse Hatası

Belirti: return () içinde root <div>'e sibling olan {/* */} yorumları TypeScript hatasına neden oluyor.

Çözüm: {/* ... */} yorumlar // ... olarak return bloğunun dışına taşındı.

Bakım Notları

⚠️ solidHeight Güncellenmeli

lib/theme.ts'te announcementHeight veya navHeight değiştirildiğinde, solidHeight de elle güncellenmeli: 

solidHeight = announcementHeight + navHeight
             = 40 + 85 = 125 // ← bunu güncelle

⚠️ globals.css CSS Değişkenleri

styles/globals.css dosyasındaki :root CSS değişkenleri (--honey-dark, --honey vb.) lib/theme.ts → palette ile manuel senkron tutulmalıdır. Bir renk değiştiğinde her iki dosya da güncellenmelidir.
