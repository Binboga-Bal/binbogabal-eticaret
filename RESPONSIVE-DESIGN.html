Responsive Design Dokümantasyonu

Mobile-first yaklaşımla xs (375px) → 5xl (3840px / 4K) arasında tutarlı, okunabilir ve erişilebilir bir deneyim sunan responsive tasarım sistemi.

Tailwind CSS Mobile-First xs → 5xl Fluid Typography clamp() Widescreen
📐

Giriş & Prensipler

Mobile-First Yaklaşım

Tüm stiller önce mobil (375px) için yazılır, ardından büyük ekranlar için genişletilir. Tailwind'in varsayılan yönü de bu şekildedir — prefix'siz class mobil için, prefix'li class (md:, lg: vb.) daha büyük ekranlar içindir.

Temel Prensipler

  • İçerik asla 100vw genişliğinde uzamamalı — max-width ile sınırlanmalı
  • Hiçbir ekranda yatay scroll olmamalı — body'de overflow-x: hidden
  • Tüm tıklanabilir öğeler min 44×44px — iOS/Android guideline
  • iOS Safari'de input zoom olmamalı — font-size minimum 16px
  • Görseller bozulmaz — aspect-ratio korumalı, Next.js Image + sizes prop
  • Widescreen'de dengeli boşluk — clamp() ile orantılı padding
📁 Dosya Yapısı: Tüm responsive altyapısı tailwind.config.ts, styles/globals.css, components/layout/Container.tsx ve hooks/ klasöründe toplanmıştır.

Container Bileşeni

Kullanım

Tüm sayfalarda tutarlı max-width ve padding için <Container> kullanılır. max-w-7xl mx-auto px-4 sm:px-6 lg:px-8 kalıbının yerini alır.

// components/layout/Container.tsx
type ContainerSize = "content" | "wide" | "ultrawide" | "full"

// content   → max-w-content  (1280px) ← varsayılan
// wide      → max-w-wide     (1536px)
// ultrawide → max-w-ultrawide(1920px)
// full      → max-w-full     (100%)

// Padding skalası:
// xs/sm  → px-4   (16px)
// md     → px-6   (24px)
// lg     → px-8   (32px)
// xl     → px-12  (48px)
// 3xl    → px-16  (64px)  ← Full HD
// 4xl    → px-24  (96px)  ← 2K
// 5xl    → px-32  (128px) ← 4K
✓ Kural: Asla max-w-7xl mx-auto px-4 sm:px-6 lg:px-8 yazmayın — bunun yerine <Container> kullanın.

Responsive Hooks

⚠️ Kural: Hook'lar yalnızca JS'te breakpoint bazlı davranış değişikliği için kullanılır. Görsel responsive için her zaman Tailwind sınıfları tercih edilmeli.

useMediaQuery

// SSR-safe — sunucuda her zaman false döner
const isMobile = useMediaQuery("(max-width: 768px)")

useBreakpoint

// Mevcut breakpoint adını döner
const bp = useBreakpoint()
// "xs" | "sm" | "md" | "lg" | "xl" | "2xl" | "3xl" | "4xl" | "5xl"

const isMobile     = ["xs", "sm"].includes(bp)
const isWidescreen = ["3xl", "4xl", "5xl"].includes(bp)

useScrollPosition

const scrollY = useScrollPosition()
// Sayfa Y scroll pozisyonu (px) — SSR'da 0

useWindowSize

const { width, height } = useWindowSize()
// Pencere boyutu — SSR'da {width: 0, height: 0}

Dokunma Hedefleri (Accessibility)

✓ Kural: Tüm tıklanabilir öğeler minimum 44×44px (iOS/Android HIG guideline, WCAG 2.5.5)
  • Button bileşeni: min-h-[44px] tüm boyutlarda
  • Input bileşeni: min-h-[44px]
  • Nav linkleri: py-2.5 ile dokunma alanı genişletildi
  • İkon butonlar: p-2.5 (40px+ tıklama alanı)
  • CSS'de button, a, [role="button"] için global min-height: 44px
  • Paragraf içi linklerde bu kural override edilir (display: inline)
↔️

Overflow & Scroll

  • Yatay scroll asla olmamalı — html, body { overflow-x: hidden }
  • Uzun tablolar: .table-responsive wrapper (overflow-x-auto)
  • Hesabım mobil nav: overflow-x: auto + minWidth: max-content
  • Modal/drawer: overflow-y-auto max-h-[90vh]
  • Carousel: overflow-hidden + scroll-snap
🖥️

Widescreen (3xl – 5xl)

Temel Kurallar

  • İçerik asla 100vw genişliğinde uzamaz — Container max-width ile sınırlanır
  • Kenar boşlukları orantılı büyür: 3xl:px-16 → 4xl:px-24 → 5xl:px-32
  • Grid kolonları artar: 3xl:grid-cols-5 → 4xl:grid-cols-6 → 5xl:grid-cols-8
  • Font boyutları fluid scale ile okunabilir kalır
  • Hero banner 3xl:aspect-[21/9] ile sinematik oran
  • Footer 5 kolona çıkar, sosyal alanı genişler
  • Müşteri paneli sidebar w-64'e genişler

iOS Safari Sorunları & Çözümleri

Input Zoom Sorunu

iOS Safari, font-size < 16px olan input'lara focus olunca sayfayı zoom'lar.

/* globals.css — global fix */
input, select, textarea {
  font-size: max(16px, 1rem);
}

/* Input bileşeni */
"text-[16px] sm:text-sm"  ← mobilde 16px, desktop'ta küçük

Safe Area (Notch / Home Bar)

/* Mobil sticky bottom bar */
paddingBottom: "env(safe-area-inset-bottom, 0px)"

Momentum Scroll

/* globals.css */
body { -webkit-overflow-scrolling: touch; }
