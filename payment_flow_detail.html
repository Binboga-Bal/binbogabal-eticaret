PCI-DSS Uyumu

SAQ A Nedir?

PCI-DSS (Payment Card Industry Data Security Standard) kapsamında, kart verisini hiç işlemeyen, saklamayan veya iletmeyen merchant'lar SAQ A kategorisindedir. Bu kategori en düşük denetim yükünü taşır. Merchant'ın tek sorumluluğu, ödeme sayfasının güvenli bir kaynaktan sunulmasıdır.

DurumAçıklamaBu Proje
Kart numarası sunucuda Yüksek risk — SAQ D kapsamı ✓ Hiç gitmez
CVV sunucuda Kesinlikle yasak — hiçbir zaman saklanamaz ✓ Hiç gitmez
app_secret tarayıcıda Yüksek risk — hash manipülasyona açık olur ✓ Sunucuda kalır
Hash key sunucuda hesaplanır Gerekli — app_secret gizli kalmalı ✓ AES-256-CBC
Form doğrudan bankaya POST SAQ A için zorunlu ✓ paySmart3D

Adım 1 — Müşteri Ödeme Formunu Doldurur

Teslimat Bilgileri

Ad, soyad, e-posta, telefon, şehir, ilçe, adres — bu bilgiler API'ye gönderilir.

Kart Bilgileri — Yalnızca Browser State'inde

Kart alanları useState<CardData> ile tutulur. React formu submit edildiğinde bu değerler API body'sine eklenmez. 

// CheckoutForm.tsx — Kart state'i
const [card, setCard] = useState<CardData>({
  cc_holder_name: "",
  cc_no:          "",
  expiry_month:   "",
  expiry_year:    "",
  cvv:            "",
});

// API'ye gönderilen body — kart verisi YOK
const body = {
  shippingAddress: data,   // ad, soyad, adres…
  items:           [...],  // variantId, quantity, price…
  subtotal, shippingFee, discount, total,
  couponCode, notes, paymentMethod,
  // ❌ cc_no, cvv, expiry burada YOK
};

Client-Side Kart Doğrulaması

  • Kart numarası: 13-19 hane, yalnızca rakam
  • Ay: 01–12 arası
  • Yıl: 2 veya 4 hane rakam
  • CVV: 3 veya 4 hane
  • Kart sahibi adı: boş olamaz

Adım 2 — API Route: Sipariş ve Token

POST /api/orders

  1. Zod şemasıyla body doğrulanır (kart alanı şemada tanımlı değil)
  2. Sipariş DB'ye PENDING durumunda kaydedilir
  3. Kupon varsa usedCount artırılır
  4. QNBPayAdapter'ın createPayment() çağrılır
  5. formPayload tarayıcıya döndürülür

QNBPay Token Alımı (Server-Side)

app_id + app_secret ile portal.qnbpay.com.tr/ccpayment/api/token'a POST yapılır. Dönen JWT, Authorization: Bearer <token> olarak formPayload'a eklenir. 

// lib/payment/qnb-pay.ts
const response = await axios.post(
  `${this.baseUrl}/ccpayment/api/token`,
  { app_id: this.appId, app_secret: this.appSecret },
  { timeout: 10000 },
);
// Yanıt yapısı: { status_code: 100, data: { token: "eyJ..." } }
const token = response.data?.data?.token ?? response.data?.token;
⚠️ DİKKAT: app_secret yalnızca sunucuda kullanılır. Tarayıcıya hiçbir zaman gönderilmez. NEXT_PUBLIC_ prefix'iyle tanımlanmamalıdır.

Adım 3 — Hash Key Üretimi (AES-256-CBC)

Hash'e Dahil Olan Veri

total | installments | currency_code | merchant_key | invoice_id
// Örnek: "199.00|1|TRY|$2y$10$...|BB-MPN2V12A-8GWU"

Adımlar

  1. iv → SHA1(random) → hex → ilk 16 karakter
  2. salt → SHA1(random) → hex → ilk 4 karakter
  3. password → SHA1(app_secret) → hex
  4. key → SHA256(password + salt) → hex → ilk 32 karakter
  5. AES-256-CBC şifreleme (key + iv ile)
  6. Sonuç: iv:salt:base64_encrypted, / → __
🔒 GÜVENLİK: Hash formülünde kart verisi yoktur. Hash yalnızca tutarın ve sipariş numarasının değiştirilmediğini doğrular. app_secret hash çıktısından geri çözülemez.

Adım 4 — Tarayıcıdan Direkt QNBPay POST

Nasıl Çalışır?

Sunucudan gelen formPayload + tarayıcı state'indeki kart verisi, JS ile DOM'a eklenen gizli bir <form>'a doldurulur. Form form.submit() ile QNBPay endpoint'ine gönderilir. Bu noktada tarayıcı QNBPay'in sayfasına geçer — merchant sunucusu bu isteği görmez. 

submitToQNBPay(payload, card) {
  const form = document.createElement("form");
  form.method = "POST";
  form.action = payload.endpoint;
  // → https://portal.qnbpay.com.tr/ccpayment/api/paySmart3D

  const fields = {
    authorization:          payload.authorization,     // Bearer JWT
    merchant_key:           payload.merchant_key,
    pos_id:                 payload.pos_id,            // QNB_PAY_POS_ID env
    currency_code:          payload.currency_code,     // TRY
    installments_number:    payload.installments_number, // 1
    invoice_id:             payload.invoice_id,        // BB-XXXXXX
    invoice_description:    payload.invoice_description,
    name:                   payload.name,
    surname:                payload.surname,
    total:                  payload.total,             // "199.00"
    items:                  payload.items,             // JSON string
    ip:                     payload.ip,
    transaction_type:       "Auth",
    is_comission_from_user: "2",
    response_method:        "GET",
    return_url:             payload.return_url,
    cancel_url:             payload.cancel_url,
    hash_key:               payload.hash_key,
    // ── Kart alanları — yalnızca buraya gelir, sunucuya uğramaz ──
    cc_holder_name:         card.cc_holder_name,
    cc_no:                  card.cc_no.replace(/\s/g, ""),
    expiry_month:           card.expiry_month,
    expiry_year:            card.expiry_year,
    cvv:                    card.cvv,
  };

  // Gizli input'ları form'a ekle, submit et
  form.submit();
}

Adım 5 — 3D Secure Akışı

QNBPay'in Rolü

  1. Form verilerini alır, hash_key'i doğrular
  2. Kartın BIN numarasından bankayı tespit eder
  3. Kartın bankasının 3D Secure sayfasına yönlendirir
  4. Müşteri SMS/OTP kodunu girer
  5. Banka doğrular, QNBPay ödemeyi işler
  6. Sonucu return_url veya cancel_url'e yönlendirir
ℹ️ response_method: GET olarak ayarlandığında QNBPay sonucu query param olarak taşır. POST olarak ayarlandığında form POST eder (production için önerilir). Şu an localhost testi için GET kullanılıyor.

Adım 7 — İptal / Hata Akışı

GET /odeme/hata — cancel_url

QNBPay ödeme başarısız olduğunda veya müşteri iptal ettiğinde bu sayfaya yönlendirir. Server component olarak çalışır: query param'lardan invoice_id ile PaymentTransaction kaydı FAILED yapılır.
